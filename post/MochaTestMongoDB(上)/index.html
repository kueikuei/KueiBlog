server.js

測試程式碼在此

主要檔案包含server.js與server.test.js
為避免篇幅過於龐大
本篇先專注介紹server.js部份。

前面有講解到如何透過node.js藉由原生的語法與MongoDB進行CRUD - MongoDB(2)

而後我們介紹透過mongoose套件進行更容易的互動，像是如何去Create資料 - Mongoose

本篇將介紹如何透過mongoose進行完整的CRUD
有意思的是
過往我們要進行的前後端完整的功能測試
我們可能前端要寫出一個簡單框架
包含按鈕還有相關direct頁面
去進行request和response的取得
然而在這邊
我們透過postman去取代前端的req發送
並且取得相對應的res結果。

一個 完整(req,res)流程 說明如下：

前端使用者按出送出鍵後產生請求(request)到伺服器端
伺服器端根據請求到DB中撈取資料並回傳(response)結果
根據這些過程從中作測試
同時測試前端與後端
前端測試req後的網路狀態是否正確、res結果是否符合預期、DB資料有無正確產生並符合預期

//create
app.post('/todos', (req, res) => {
  var todo = new Todo({
    text: req.body.text
  });

  todo.save().then((doc) => {
    res.send(doc);
  }, (e) => {
    res.status(400).send(e);
  });
});

postman將method設定為post

並且創造一個新todo物件

並且藉由save()儲存入DB中

在Todo Project中
我們有提過subredditName的概念
這邊我們再以相同技術來看一個例子

var ObjectID = require('mongodb').ObjectID;

app.get('/todos/:id', (req, res) => {
  var id = req.params.id;

  if (!ObjectID.isValid(id)) {
    return res.status(404).send();
  }

  Todo.findById(id).then((todo) => {
    if (!todo) {
      return res.status(404).send();
    }

    res.send({todo});
  }).catch((e) => {
    res.status(400).send();
  });
});

網址列輸入/:id 後
我們可以藉由req.params.id去取得
下一步我們進行判斷
判斷其是否為正確並存在的id

引入require('mongodb').ObjectID
透過其方法isValid(id)
回傳一個boolean
non valid則會回傳res.status(404).send()
代表沒有驗證過

有驗證過要考量其是否真的還存在於DB中
故如果讀不到(!todo)
則一樣回傳return res.status(404).send();錯誤情況
此外屬於正確

附帶一提
ObjectID.isValid()用於判斷此id是否為MongoDB所產生
合理所產生回傳true反之為false

update
前提假設：
1.text不是空的(null) -> completed顯示為true
2.進行更新時要重新更換建立時間completedAt
3.否則text=null，completed = false;completedAt = null;

if (_.isBoolean(body.completed) && body.completed) {
    body.completedAt = new Date().getTime();
  } else {
    body.completed = false;
    body.completedAt = null;
  }

進行更新

Todo.findByIdAndUpdate(id, {$set: body}, {new: true})

delete

app.delete('/todos/:id', (req, res) => {
  var id = req.params.id;

  Todo.findByIdAndRemove(id).then((todo) => {
    if (!todo) {
      return res.status(404).send();
    }
    res.send({todo});
  }).catch((e) => {
    res.status(400).send();
  });
});

看完了CRU後
看D就顯得簡單
