npm

  1. $ npm init產生package.json檔
  2. $ npm install透過package.json將相關套件進行安裝，並產生node_modules資料夾存放套件
  3. 或是$ npm install --save 'package_name'將相關套件引入並且在package.json中dependencies套件
  4. $ npm install --save -dev 'package_name'會將相關套件引入並且存放在package.json中devDependencies套件

建議將dependencies與devDependencies事先寫入package.json檔案中再直接$ npm install

bower

$ bower作為前端管理的輕量級套件，實行步驟與$ npm類似。

  1. $ bower init產生bower.json檔
  2. $ bower install --save 'package_name'將相關套件引入並且在bower.json中dependencies套件，此外，會產生bower_components裡面放置相關套件

gulp

自動化建置工具gulp

$ gulp透過gulpfile.babel.js 檔案能將相關資源檔轉譯成靜態網頁所需要的檔案

透過gulpfile.babel.js轉譯成瀏覽器可以讀取的語言

sass轉成css
js優化處理，最終剩下bundle.js
image壓縮處理

在進行$ gulp時若出現gulpfile.babel.js無法讀取的狀況，可增加.babelrc檔案讓$ gulp指令能順利執行

//.babelrc
{
  "presets": ["es2015"]
}

$ gulp轉譯後產生dist 資料夾，裡頭存放轉譯後的js與css file

main.js經過gulp後轉譯成bundle.js，而專案在讀取.js檔案主要是讀取bundle.js file

push to github

  1. 產生public 資料夾存放所有遠端檔案與資源，除了node_modules外，將所有檔案皆放入此public 資料夾中

  2. 遠端github產生repository，並將此repo的GitHub Pages開啟，創立名為gh-pages的分支，切換到Settings，並在GitHub Pages中的Source切換到gh-pages並Save

  3. 本地端的檔案，所有的檔案資源路徑與相關routing都要配合遠端repo_name進行更改：

html方面(例如index.html、create.html、update.html等等)

<link rel="stylesheet" type="text/css" href="repo_name/bower_components/bootstrap/dist/css/bootstrap.css">
<link rel="stylesheet" type="text/css" href="repo_name/dist/css/main.css">

js方面(main.js與bundle.js都要改)

更改方面建議的方式是更改完main.js後再透過$ gulp自動去更改bundle.js
最重要的是routing上的更改：

switch (path) {
    case '/repo_name/create.html':
        submitListener('create');
        break;
    case '/repo_name/update.html':
        readFormData();
        submitListener('update');
        break;
    default:
        readAccountData();
}

git push to repo

  1. 本地端透過$ git checkout -b gh-pages產生名為gh-pages的分支`
  2. 逐步實施$ git init、$ git add file_name、$ git commit -m'description'後
    透過$ git push origin gh-pages上傳到遠端分支上
  3. 不幸出現failed to push some refs to...的話，透過$ git pull --rebase origin gh-pages合併本地端與遠端，並再次$ git push origin gh-pages
  4. 順利的話就能透過github產生靜態分頁，並透過此網址https://kueikuei.github.io/repo_name/進行網站瀏覽
