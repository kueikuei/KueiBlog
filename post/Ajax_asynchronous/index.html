Ajax介紹

全名為Asynchronous JavaScript and XML，中文譯為非同步的JavaScript與XML技術，然而現今不只單純用XML，JSON更是一大主流。是一套綜合了多項技術的瀏覽器端網頁開發技術。可以完成不刷頁局部更新應用，使用者體驗較好。

同步Synchronous與非同步Asynchronous

  • 同步就是要等到上一個函數任務執行完才能執行下一個，是依序執行
  • 非同步係指程式不會因為上一個函數尚未執行完(例如:回傳結果)就卡住，會往下執行下一個任務

傳統的http同步Synchronous動作，user對browser發送一個requset後server再回傳一個response，如此便停止動作，直到user再度發送一個requset，這些順序是循序漸進，開始結束開始結束如此循環，但是缺點在於，當user的網路過慢時，可能會看見網路空白頁以及逐漸載入的過程，影響使用者觀感。

Promise

1. Promise為何而生？
在開發時，我想或多或少都會碰到以下情況，多個接口非同步請求數據時，每个接口的調用都得上一個接口的结果，這就是有名的迴調金字塔，用程式碼顯示如下：

    $.ajax({
    url: '/a',
    success: function(data) {
        if (data.code === 1) {
            $.ajax({
                url: '/b',
                success: function(data) {
                    if (data.code === 1) {
                        $.ajax({
                            url: '/c',
                            success: function(data) {
                                // ......
                            }
                        });
                    }
                }
            });
        }
    }
});

而Promise則是為了避免非同步處理進入廻調地獄callback hell的處理方法之一。

請不要誤會，Promise並不是Javascript特有的處理方式，而是像諸如OOP、演算法等，是每一種程式語言都有相似的處理方式，比如：

  • jQuery的Deferred
  • angular的$q
  • Java的java.util.concurrent.Futrue
  • Python的Twisted deferreds and PEP-3148 futures
  • F#的Async
  • .Net的Task
  • Javascript的Promise A+

2. 细說Promise
同步執行函式的結果要不就是回傳一個值，要不然就是執行到一半發生例外，中斷目前
的程式然後拋出例外。
異步的函式結果又會是什麼？要不然就最後回傳一個值，要不然就執行到一半發生例外，但
是異步的函式發生錯誤時怎麼辦？可以馬上中斷程式然後拋出例外嗎？不行。那該怎麼作？
只能用別的方式來處理。也就是說異步的函式，除了與同步函式執行方式不同，它們對於錯
誤的處理方式也要用不同的方式。

簡單來說，異步執行函式的結果要不就是帶有回傳值的成功，要不就是帶有回傳理由(reason)的失敗。

Promise的狀態

從Promise模式的角度看，完成一個任務，可分成3種狀態。

  • pending - 未完成，準備狀態
  • resolve - 任務成功時調用（已完成，又稱Fulfilled）
  • reject - 任務失敗時調用

Promise的狀態變化只發生一次(從pending變成resolve或reject)，變化後狀態凝固。

Promise的API
由ECMAScript所定義，最普遍有以下兩種：

  • Constructor
  • Instance Method

和一些內建物件一樣(例如Date)，這邊或能將Promise想像為物件較容易理解，我們可以用Constructor 来產生一個Promise:

var promise = new Promise(function(resolve, reject) {
    //處理非同步,结束後調用resolve或reject
});

物件都有內建的方法，Promise也不例外，最常見的為then、catch方法，延續剛剛的例子，在產生一筆新的Promise Object後，接著無非是想知道成功(resolve)取得資料或失敗(reject)：

promise.then(function onResolve() {
    //成功時調用
}.catch(function onReject(error) {
    //失敗時調用
});

easy example

//define data
const comments=[
  {title:'Comment1',content:'content1'},
  {title:'Comment2',content:'content2'},
];

function getComments(){
  return new Promise(function(resolve,reject){
    setTimeout(function(){
      resolve(comments);
    },1000);
  });
}

function printCommentsToConsole(){
  getComments()
  .then(comments=>console.log(comments))
  .catch(err=>console.log(err));
}

printCommentsToConsole();

ajax promise

var getJSON = function(url) {
  var promise = new Promise(function(resolve, reject){
    var client = new XMLHttpRequest();
    client.open("GET", url);
    client.onreadystatechange = handler;
    client.responseType = "json";
    client.setRequestHeader("Accept", "application/json");
    client.send();

    function handler() {
      if (this.readyState !== 4) {
        return;
      }
      if (this.status === 200) {
        resolve(this.response);
      } else {
        reject(new Error(this.statusText));
      }
    };
  });

  return promise;
};

getJSON("http://163.29.157.32:8080/dataset/6a3e862a-e1cb-4e44-b989-d35609559463/resource/f4a75ba9-7721-4363-884d-c3820b0b917c/download/393625397fc043188a3f8237c1da1c6f.json'").then(function(json) {
  console.log('Contents: ' + json);
}, function(error) {
  console.error('出錯了', error);
});

fetch API

讓原本的XMLHttpRequest更簡單
fetch() 主要使用兩個參數,請求地址和選項。請求完成後return Promise 物件

這邊我們用剛剛的ajax promise為例子，透過fetch去做做看
範例如下：

const myHeaders = new Headers();

if(self.fetch){
//runmyfetchrequesthere
const myInit={
    method:'GET',
    headers:myHeaders,
    mode:'cors',
    cache:'default'
};

const REQUEST_URL='http://163.29.157.32:8080/dataset/6a3e862a-e1cb-4e44-b989-d35609559463/resource/f4a75ba9-7721-4363-884d-c3820b0b917c/download/393625397fc043188a3f8237c1da1c6f.json';

//do fetch API
fetch(REQUEST_URL,myInit)
    .then(function(response){
        return response.json();
    })
    .then(function(data){
        console.log(data);
    });

}else{
//dosomethingwithXMLHttpRequest?
}
