前言

在上一篇MongoDB(2)中有提到原生的Mongo如何透過local端套件Client寫入資料到DB
而透過Mongoose可以更容易更直觀地寫入，此外，最大的好處在於可以定義Schma以及Validation
類似於關聯資料庫中的實體(entity)定義

Schma and Validation

透過定義Schame，賦予資料準則
例如password document
我們不會指定字串String作為password
而Schama就在幫我們定義這些規則並加以驗證(Validation)
不遵照規則則無法產生資料進入資料庫
除了data type外，還能指定長度、預設值等等

以下為簡單例子：

text: {
  type: String,
  required: true,
  minlength: 1,
  trim: true
},
completed: {
  type: Boolean,
  default: false
},
completedAt: {
  type: Number,
  default: null
}

詳細請看Mongoose Schema

Mongoose連結資料庫
mocha_express/n-7-13-schemas/server/server.js

var User = mongoose.model('User', {
  email: {
    type: String,
    required: true,
    trim: true,
    minlength: 1
  }
});

var user = new User({
  email: 'andrew@example.com   '
});

user.save().then((doc) => {
  console.log('User saved', doc);
}, (e) => {
  console.log('Unable to save user', e);
});

mongoose.model可以想像為烘焙時餅乾形狀容器
產生出一個名為User的容器
並定義出容器的形狀與規則Schema
其中trim代表不能有長空格

要創造出容器內容物使用物件呼叫法
new User
這邊看到的email資料為必填required
後面的空格因為有trim
在寫入DB後會自動往前縮排
最後user.save()存入DB中

Server.js - post and create data by webfront
mocha_express/n-7-15-post-todos/server/server.js

var express = require('express');
var bodyParser = require('body-parser');

var mongoose = require('./db/mongoose');
var Todo = require('./models/todo');
var User = require('./models/user');

var app = express();

app.use(bodyParser.json());

結合express進行應用
express能夠輕易建立request url direction
詳細express用法也能參考範ToDoproject

重要的是，搭配body-parser
如此便能從前端的body中去取得某些資訊
app.use(bodyParser.json())扮演重要的資料轉換主角
在此範例
我們將取到的資訊加以存入DB

//------post--------
//todos
app.post('/todos', (req, res) => {
  var todo = new Todo({
    text: req.body.text
  });

  todo.save().then((doc) => {
    res.send(doc);
  }, (e) => {
    res.status(400).send(e);
  });
});

//user
app.post('/users', (req, res) => {
  var user = new User({
    email: req.body.email
  });

  user.save().then((doc) => {
    res.send(doc);
  }, (e) => {
    res.status(400).send(e);
  });
});

重頭戲來了
要create資料則使用post method
所以這邊使用app.post
new一個新的data
req.body幫我們取得前端資料
這邊的email(key)必須固定
因為我們在model已經定好
接著就是postman使用

Postman使用


步驟
1.選擇post
2.網址貼上localhost:3000/collection name
3.選擇body->raw->json
4.打入req資料
5.Send送出

順利的話下面就會跑出結果
並顯現status:200 OK與資料
並傳入MongoDB中
