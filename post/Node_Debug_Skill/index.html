Debug Mode

node版本要在v8.1.2以上
可以透過$ node -v查詢
接著在cmd打入node inspect js.file進入debug mode

在debug mode中
可以透過list(line num)查詢file內的程式碼

n & repl

打入n會一次執行三行，再打入，則循序漸進直到將完整程式碼跑完
跑完後打入repl（Read-Eval-Print Loop）
則可以像是chrome dev cmd去執行程式裡頭變數的操作與觀察

像是nodejs里的repl裡頭介紹的：
如果用过rails
一定很喜欢rails console
它可以直接调用AR里的各种方法
非常简单的就可以测试对db的各种操作。

需求

  • 加载app/models下的所有mongoose模型
  • .list命令打印出所有的可用实体
  • 执行User.find({},function(err,doc){console.log(doc)})可以查出具体内容

c & debugger

c代表cont
會一次執行完所有程式
但執行完後要再重跑run運行程式
也不能進入repl模式

而透過debugger
在要觀察的變數或函數插入進行斷行
$ c後會執行到debugger處
再進入repl模式則可以查看
比起n更便利與快速
