Bootstrap介紹

學習Bootstrap就如同國高中的歷史地理背誦，是一種洗腦，一種毒。

Bootstrap讓前端工程師能夠快速開發出好看的頁面，也無需執著於雕刻CSS，很方便沒錯，但缺點在於，大家刻出來的東西都大同小異、此外，必須照著其規則走，故剛開始接觸時，會感覺不像在寫程式，比較像是在背誦程式。

Bootstrap RWD應用

RWD(Responsive web design)稱自適應網頁設計、回應式網頁設計、對應式網頁設計。 是一種網頁設計的技術做法，該設計可使網站在多種瀏覽裝置(維基百科)。

本節介紹一些Bootstrap內建的RWD關鍵字：

  • jumbotron
  • container
  • img-responsive

Bootstrap的jumbotron超大屏幕。可以增加標題的大小，為頁面内容添加更多的margin
container提供一個共用的RWD佈局方式
那麼圖片的RWD呢？別擔心，Bootstrap也預料到了，只要加上img-responsive就能輕鬆做到了

<div class="jumbotron">
  <div class="container">
    <h1>Hello Bootstrap!</h1>
    <p>bootstrap是個很多人使用的網頁模板，裡面有很多別人寫好的現成css資源，也有用來排版很方便的網格系統。</p><a class="btn btn-primary" href="https://kkbruce.tw/bs3/" target="_blank">了解更多(中文說明文件)</a>
  </div>
</div>

Bootstrap 常用基礎元件

  • Button

  • list-group/list-group-item

前面已經介紹過Button，這邊我們來聊聊其他好用的Bootstrap元件吧:))

Breadcrumb
Breadcrumb中文翻譯為麵包屑，麵包屑一層一層疊加一起，適合用來表達有遞呈關連的當前網頁瀏覽位置
加上breadcrumb即可使用，然而在hexo的markdown筆記下無法順利產生，原因尚在查詢:(

<h3 class="page-header">麵包屑導覽列</h3>
    <ul class="breadcrumb">
    <li> <a>動態網頁課程</a></li>
    <li>bootstrap介紹</li>
    <li>第一個範例</li>
</ul>

list
加上list-group定義清單群組，接著每一個子清單都要加入list-group-item關鍵字。
而清單有著諸多功能，包含加入disabled此清單就無法選取
加入badge還有提示的功能，配合JS改變數字變數，就可以進行一些計算，例如留言數等等，在當前網頁中經常看到。

<h3 class="page-header">清單練習</h3>
  <ul class="list-group">
    <li class="list-group-item active">list-group 清單<span class="badge">15</span></li>
    <li class="list-group-item disabled">btn btn-default 按鈕</li>
    <li class="list-group-item list-group-item-success">breadcrumb 麵包屑尋覽列</li>
  </ul>



  • list-group 清單15

  • btn btn-default 按鈕

  • breadcrumb 麵包屑尋覽列


table
使用起來相當簡單，只要加入table關鍵字即大功告成。

Bootstrap格線系統

12column compose 1row, and rows be consist of FrontWeb

Bootstrap將螢幕寬度分為12等分(12 column)，而在定義column前必須先定義row。
故標準公式為：

<div class="row">
    <div class="col-xs-1">1</div>
    <div class="col-xs-1">2</div>
    <div class="col-xs-1">3</div>
    <div class="col-xs-1">4</div>
    <div class="col-xs-1">5</div>
    <div class="col-xs-1">6</div>
    <div class="col-xs-1">7</div>
    <div class="col-xs-1">8</div>
    <div class="col-xs-1">9</div>
    <div class="col-xs-1">10</div>
    <div class="col-xs-1">11</div>
    <div class="col-xs-1">12</div>
</div>

按照尺寸來分可以分為4(xs/sm/md/lg)個尺寸，讓我們看看上面的例子可以發現使用的是xs超小螢幕尺寸，這是因為尺寸是向上覆蓋的(比如如果指定md中螢幕佔7欄，沒有指定lg大螢幕佔多少，會沿用中螢幕的分配)，故原則上都要定義xs，否則板面容易跑掉，或是透過visible-size、hidden-size關鍵字在特定尺寸下顯示或隱藏，如此便完成RWD的設計。

完整範例參考Demo 8-7 RWD原理與應用-bootstrap 格線系統原理與RWD應用
