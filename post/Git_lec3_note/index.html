合併分支(git merge)

今天不管在一個分支上做了多少開發，終究還是需要把該分支和預設分支 (master) 進行合併，這樣才會把分支上的 commit 納入 master 裡面

情境課題:當不同分支的同一檔案剛好有不同的程式碼 

//lesson3_1 commit
$echo '# hello world!' >> lesson3_1.md
$git add lesson3_1.md
$git commit -m "added lesson3_1.md"

//lesson3_2 commit
$echo '# hello world!' >> lesson3_2.md
$git add lesson3_2.md
$git commit -m "added lesson3_2.md"

//new branch feature
$git branch feature
$git checkout feature

//現在我們在 feature branch 上做一個 commit...
$echo '# another hello world!' >> lesson3_1.md
$git add lesson3_1.md
$git commit -m "edit lesson3_1.md"

//現在我們回到 master branch 上：
$git checkout master

//接下來我們再創造一個檔案並且做 commit:
$echo '# hello world in lesson3_3.md' >> lesson3_3.md
$git add lesson3_3.md
$git commit -m "added lesson3_3.md"

//此時 master 與 feature 的內容開始出現 "分叉"

//接下來把 feature branch 上面的 commit 合併到 master branch 上：
$git merge feature 

//此時我們已經完成合併，也就是把 feature 分支裡的內容加入 master 分支裡

注意merge並非是把feature分支的commit放入master內
而是在master上創造一個新的'merge commit'
裡面包含了所有 feature 分支的內容

git merge再深入

merge包含
(1) fast-forward merge
(2) 3-way merge

接著我們來詳細討論此兩種差異

fast-forward merge
如果想要合併的提交可以直接往回追溯歷史到目前所在的提交
也就是說branch的內容只是原來檔案的增加或減少
Git 會因為沒有需要合併的工作而簡單地把指標向前推進
這就是所謂的「快進（fast-forward）」。

example

//master file
$echo '# hello world!' >> master.md
$git add master.md
$git commit -m "added master.md"

//此時的master file只有'# hello world!'

//branch file
$git branch feature
$git checkout feature
$echo '# hello branch!' >> branch.md
$git add branch.md
$git commit -m "added branch.md"

//此時的branch file有'# hello world!# hello branch!'

$git checkout master
$git merge feature

//此時就會進行fast-forward



git merge ‘branch_name’
將head移動到原來的branch再併吞後來延伸的，稱為
fast-forward merge
不會留下merge的紀錄

git merge –no-ff ‘branch_name’
—no
3-way merge
會留下merge記錄

git lg2 從cmd去查看commit graph

案例情形：遠端與本地端的commit tree 不同時處理辦法

(1) git fetch - -all 更新 “所有” remote 底下的分支
等同於 git remote update
(2) git pull ‘remote name’ ‘brench name’
同步本地端分支（Merge）
等同於
$ git fetch ‘repo_name’
$ git merge remotes/github/master

(2)-(2) git pull - -rebase
將遠端資料更新至local
並且添加遠端commit到local端主線
減少分之產生
使commit更精簡

####git rebase master
