Gulp介紹

Gulp能處理網頁自動化流程，所謂的自動化流程，概念是將某些工作自動化去執行，有點類似模組的概念，不用每一次跑一個專案都做同樣的事情。

那麼，哪些事情是需要自動化流程的呢?
筆者認為大概能區分為編譯檔案、壓縮檔案、部屬動作。
有了這些自動化流程，在開發時能更增加效率、壓縮後得檔案在部屬後能更快得被載入網頁中。

壓縮優化檔案

無論是Html、CSS、還是Javascript，程序人員編寫時都會透過縮排或特定格式讓其好觀看、好編寫，而無形中也產生許多不必要的空格，挪用了資源空間。
其中，圖片更是深深影響使用網頁載入的速度，故透過gulp去壓縮這些檔案將使整體網頁更流暢快速，增加使用者經驗好感。

主要使用到的套件如下:

gulp-jade:壓縮jade，也能進一步增加參數pretty: true使其恢復為好讀排版。
gulp-minify-css:壓縮css
gulp-uglify:壓縮JS，進一步透過compress:{drop_console: true}能自動移除掉console.log
gulp-imagemin:壓縮圖片檔

更進一步，我們能決定哪些環境時要壓縮那些不用壓縮。
例如當還是完整成品production時，我們稱為develop environmention
此時我能能夠過兩個套件來定義這些環境，並藉由傳入參數來決定是否壓縮

minimist與gulp-if套件:創造不同開發環境develop和production
而由於圖片壓縮會浩比較多的時間，適合在production時在壓縮

使用範例如下:

透過$gulp --env {參數}就能將參數帶入至option
process.argv.slice(2) 這行則是跳過前兩個 – 字元，直接讀取 env 這個字串

而透過$.if(options.env==='production', $.minifyCss())進行輸入參數的判斷
即可得到壓縮成甚麼樣的版本

environment choice

前面有提過不同的開發環境，反過來說，今天從git clone某個專案到本地端時，或是要上傳到heroku時都要選擇開發環境。

如果要單純就純測試環境dev進行安裝可以額外增加-dev如下:
devolpe-eni:npm install package_name –save-dev
而一般的pcakage save如下:
production-eni:npm install package_name –save

安裝特定環境可以如下指令:
npm install --only=dev
npm install --only=prod

將dev環境下的package移除，只保留production的
npm prune –prod

tips

1.soucemap()可以用來追蹤檔案來源

2.不希望語法錯誤而造成監聽(watching)停止時
透過plumber()去預防

3.透過gulp-load-plugins簡化gulp載入流程
所有gulp套件都能透過統一變數，例如$進行集中使用

4.PostCSS 與 autoPrefixer
CSS常常有瀏覽器版本不支援某些特效的問題，或是要增加前綴詞讓瀏覽器支援，而透過PostCSS與autoPrefixer自動去查詢瀏覽器支援度並加入適當前綴詞

5.gulp有其自己的測試local端-browser-sync，並搭配browserSync.stream()自動做網頁重新整理

6.build提交要上架的專案
包含gulp-clean與gulp-sequence
gulp-sequence透過clean將public移除後再依序重新產生檔案
過程中記得加入--env production來產生最小化的檔案
