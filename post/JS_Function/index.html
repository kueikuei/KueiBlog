迭代函數map()/filter()/reduce()
又稱為functional programing
一般來說用在將array中的value迭代出來，取代原本的forloop操作

迭代方法map()

const numbers=[1,2,3,4];

const newNumbers=numbers.map(function(number,index){
  return number*2;
});

console.log("The double dnumbers are",newNumbers);//[2,4,6,8]

最後的結果也是陣列。

如果不用map()去做，用傳統的迴圈來跑如下：

for(var i=0;i<numbers.length;i++){
  numbers[i]=numbers[i]*2;
}
console.log("The double dnumbers are",newNumbers);

是不是覺得map更直觀呢？

過濾方法filter()

const numbers=[1,2,3,4];

const newNumbers=numbers.filter(function(number){
  return(number%2!==0);
}).map(function(number){
  return number*2;
});

console.log("The doubled numbers are",newNumbers);//[2,6]

累計方法reduce()

var numbers=[1,2,3,4];
var totalNumber=numbers.reduce(function(total,number){
    return total+number;
},0);
console.log("Thetotalnumberis",totalNumber);//10

以上都是在處理陣列時常用的方法。

自調用函數Self-Invoking Function
Immediately‐Invoked Function Expression(IIFE)
中文稱為自調用函數
保護變數不受污染
使用方式為先解讀裡面的匿名函式後，後面再加個()去執行它。

example

(function(name){
    var cat=name;
    document.write(cat);
})('momo');

jQuery經常用到自調用函數
其主要功用是簡化JS DOM，讓開發者簡單去操作DOM

var hoisting

JS中，函數在被執行之前就會先被解析(hoisted)，因此變數可以函數內任意的地方
宣告，先宣告卻未被定義的變數則會被設定為undefined。

由剛剛的例子得知，JavaScript是function scope，無論你在函數內那裏宣告變數
var，都會提升到最前面宣告，稱為變數的拉升(Variable Hoisting)。

建議把變數的宣告放在函數的最頂端。否則可能導致混亂的情況，或是使用let可避免這種狀況
詳細用法可以參考JS_ES6_Variable此篇文章

example1

function func(){
    console.log(a);//儘管還沒定義但會回傳 undefinded
    var a=1;
}

func();

example2

//首先，Global Execution Context（A wrapper to help manage the code that is running）會被建立
//function a 和 b 由於 hoisting 的緣故會先被建立在記憶體中，接著才會開始逐行執行程式。
function b(){};
function a(){
    function b(){};
};
//接著，程式碼會執行到 a( ) 這個部分，這時候，會建立a的execution context，並且被放置到execution stack中。
//由於在 function a( ) 裡面有去執行到 function b ，因此，在這個execution stack中，最上面會變成 function b 的execution context。
//當 function b 執行完之後，會從execution stack中離開，繼續逐行執行function a。
//當 function a 執行完之後，一樣會從execution stack中抽離，在回到Global Execution Context逐行執行。
a();


//變數的抬升:無論在函數哪裡宣告變數var，都會提升到最前面宣告
//JS是function scope
//建議使用let宣告變數
function fuc(){
    console.log(a);//會顯示undefined，儘管還沒宣告
    var a=1;
}

Scope Chain

用var宣告的變數，作用範圍在當時的環境(函數內);沒有用var宣告則是全域物件(window)的屬性，也就是全域範圍。
scope chain特性，JS在查找變數時，會循著範圍練一層層往外找，函數內找不到則往函數外找
內部函式可以存取外部函式變數，外部則無法存取內部。

exercise:攝氏轉華氏function

function transfer(c){
    return (9.0*c)/5.0+32.0;
} 

console.log(transfer(20));

Closure

閉包(Closure)是擁有閒置變數(Free variable)的物件。
閒置變數既不是區域變數(local var)也不是傳入function的參數(params)。
建立函數不等於建立閉包。如果函數的閒置變數與當時語彙環境綁定，該函數才稱為閉包。
能將外部函式的變數關到內部函式，用途是延續外部變數的生命。
閉包所關起來的是變數的reference，而不是變數的value。

Example1

先建立起greet()，而一旦greet被呼叫，便會回傳裡頭的匿名函數，並將閒置變數whattosay儲存在記憶體中。

function greet(whattosay){
    return function(name){
            console.log(whattosay+''+name);
    };
};

呼叫greet()，並儲存在var guest，此時Welcome被儲存在whattosay中，既使greet()運作完仍然存活著，延續其生命週期。

var guest = greet('Welcome');

再一次呼叫guest變數，此時能將其當作該該的匿名函數運用，把Jason傳入參數name，由於剛剛的Welcome被記錄下來，故結果印出Welcome Jason

guest('Jason');

而每一個新建的閉包都是不同記憶體，故每一個閒置變數都是不同的。

如Example2，f1與f2各呼叫doOther()並將裡頭的f()存起來，但是所存到的變數x卻是不同的，故f2再一次呼叫doOther()時，x仍然是10，-1後依舊為9。

Example2

function doOther() {
    var x = 10;
    function f() {
        x--;
        return x;
    }
    return f;
}

var f1 = doOther();
var f2 = doOther();

f1();//9
f2();//9

閉包(closure)經常應用在實現private變數、嵌套callback事件處理、非同步處理上。

實現private變數
javascript不像java或是C可以透過private關鍵字去定義私有變數
但是透過其function scope的觀念其實是能做到的。

var makeCounter = function() {
  var privateCounter = 0;
  function changeBy(val) {
    privateCounter += val;
  }
  return {
    increment: function() {
      changeBy(1);
    },
    decrement: function() {
      changeBy(-1);
    },
    value: function() {
      return privateCounter;
    }
  }  
};
console.log(privateCounter);//privateCounter is not defined
var Counter1 = makeCounter();
var Counter2 = makeCounter();
console.log(Counter1.value()); /* logs 0 */
Counter1.increment();
Counter1.increment();
console.log(Counter1.value()); /* logs 2 */
Counter1.decrement();
console.log(Counter1.value()); /* logs 1 */
console.log(Counter2.value()); /* logs 0 */
  1. 模擬JAVA與C的class
  2. privateCounter在function中類似於私有變數，故無法讀取也無法更改，如此便保護了變數
  3. 而value()則是公開管道讓我們能讀取privateCounter
  4. 每一個新創建的closure都是不同記憶體，故Counter1與Counter2是不同的

非同步處理
在JS中有許多非同步(asynchronous)的事件處理，舉凡撈取open data，這邊我們以setTimeout進行模擬，期望結果是將0~5依序印出，但是最後結果卻是全都是5，原因何在？在這等待的1秒中，迴圈已經非同步的跑到了5，最後連續印出五個5。

for(var i=0;i<5;i++){
    setTimeout(function(){console.log(i);},1000);
}

顯然的，剛剛的結果並不是我們要的，於是透過closure解決此問題，由於每一個閒置變數都存於不同記憶體，故能順利將結果印出來，依序印出0~5

function printLog(i){
    //closure
    return function(){
        console.log(i);
    }
}
for(var i=0;i<5;i++){
    setTimeout(printLog(i),1000);
}

當然，透過promise也能解決非同步問題，有興趣的讀者能讀讀Ajax asynchronous 。

//Example2
function buildFunctions(){
    var arr = [];

    //一開始 i = 0，這時候會把陣列的內容 function( ){console.log(i)} 儲存到陣列中
    //但要注意的是這時候這個被儲到陣列中的函式並沒有執行（invoke），而是只是儲存在裡面而已

    for(var i=0;i<3;i++){
        arr.push(
            function(){
                console.log(i);
            }

//>3

console.log(fs[0]);

console.log(fs[2]);

Arrow Function

箭頭函數(Arrow Function)是ES6之後可以使用的功能，讓程式碼更為簡潔。

基本語法

(param1, param2, …, paramN) => { statements } 
(param1, param2, …, paramN) => expression
// 等於 :  => { return expression; } 

// 只有一個參數時,括號才能不加:
(singleParam) => { statements }
singleParam => { statements }

//若無參數,就一定要加括號:
() => { statements }

Examples

// An empty arrow function returns undefined
let empty = () => {};

(() => "foobar")() // returns "foobar" 

var simple = a => a > 15 ? 15 : a; 
simple(16); // 15
simple(10); // 10

let max = (a, b) => a > b ? a : b;
