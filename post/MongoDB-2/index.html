前言

承襲上一篇MochaTestExpress，為了單元測試MongoDB，本篇讓我們深入了解一下MongoDB運作流程。

在MongoDB曾經聊過諸多MongoDB的相關介紹以及Mongo Shell的指令
但是諸多細節還是沒有提到
故一併在本文提。

Mongo Shell啟動及基本檢查

步驟一
終端機輸入sudo mongod並輸入passpord

步驟二
cmd+t開啟新視窗並輸入mongo開啟MongoDB shell
此時能輸入各種指令操作MongoDB

基本檢查
db檢查當前使用中的資料庫
show dbs檢查所有的資料庫
use [dbname]切換到需要使用的資料庫
show collections查看當前資料庫有哪些collections
collections對應關聯資料庫中的tables
documents對應rows
fields對應columns

Robomongo

儘管能夠透過cmd操作
但是如果覺得不夠親切
這邊提供一個好工具:Robomongo
作為介面化操作MongoDB的好工具

Robomongo操作資料庫
左上角點擊connect後，create一個新連結
在地端會以localhost:27017作為port
並給予DB名稱後新增一個MongoDB
這邊可以create collection並且進一步進行CRUD等功能

Node操作MongoDB並進行CRUD

我們的目標當然是能夠透過程式碼控制MongoDB
先將MongoDB引入專案npm install mongodb --save

程式碼講解

範例程式碼在此

操作之前一樣要開啟MongoDB shell

連結資料庫並寫入資料(Create)
n-7-5-connect-write/playground/mongodb-connect.js

//require
const MongoClient = require('mongodb').MongoClient;

MongoClient.connect('mongodb://localhost:27017/todo', (err, db) => {
  if (err) {
    return console.log('Unable to connect to MongoDB server');
  }
  console.log('Connected to MongoDB server');

  db.collection('Todos').insertOne({
    text: 'Something to do',
    completed: false
  }, (err, result) => {
    if (err) {
      return console.log('Unable to insert todo', err);
    }

    console.log(JSON.stringify(result.ops, undefined, 2));
  });
});

藉由MongoClient套件去連結local端DB
mongodb://localhost:27017/todo，todo位置為DB name，可以連結或是創造新的DB
db.collection('Todos')連結想要連結的collection
insertOne({})插入新資料並回傳結果
詳細的方法可以參考insertOne

運用ObjectId()
n-7-6-objectid/playground/mongodb-connect.js

db.collection('Users').insertOne({
    name: 'Andrew',
    age: 25,
    location: 'Philadelphia'
  }, (err, result) => {
    if (err) {
      return console.log('Unable to insert user', err);
    }
    console.log(result.ops[0]._id.getTimestamp());
});

mongodb為每一個document做出獨一無二的ID
我們可以應用此ID做諸多應用
包含查詢或是得到他的建立時間getTimestamp()

mongodb package裡頭有諸多方法，包含剛剛使用的MongoClient、ObjectId都可以在
Node.js MongoDB Driver API搜尋到

讀取(Read)資料
n-7-7-fetching-data/playground/mongodb-find.js

//test 1.
db.collection('todos').find({
    _id: new ObjectID('586b4132d1d01f38cf0da8ea')
    // "_id" : new ObjectId("586b4132d1d01f38cf0da8ea") //ObjectId is not defined
    }).toArray().then((docs) => {
    //if not toArray()
    //TypeError: db.collection(...).find(...).then is not a function
    console.log('Todos');
    console.log(JSON.stringify(docs, undefined, 2));
    }, (err) => {
    console.log('Unable to fetch todos', err);
});

//test 2.
//計算共幾筆資料
db.collection('todos').find().count().then((count) => {
    console.log(`Todos count: ${count}`);
    }, (err) => {
    console.log('Unable to fetch todos', err);
});

//test 3.
//by key:value to find
db.collection('todos').find({name: 'Andrew'}).toArray().then((docs) => {
    console.log(JSON.stringify(docs, undefined, 2));
});

此三個範例分別藉由
ObjectID、{key:value}、get all data進行搜尋
要特別注意的是id搜尋法一定要用_id: new ObjectID()進行搜尋

刪除(Delete)資料
n-7-9-deleting-documents/playground/mongodb-delete.js

//1.
// deleteMany
db.collection('Todos').deleteMany({"test" : "test"}).then((result) => {
    console.log(result);
});

//2.
// deleteOne
db.collection('Todos').deleteOne({"completed" : false}).then((result) => {
    console.log(result);
});

//3.
// findOneAndDelete
db.collection('Todos').findOneAndDelete({completed: false}).then((result) => {
    console.log(result);
});

MongoDB透過三種主要方法進行刪除
deleteMany({})、deleteOne({})、findOneAndDelete({})
裡面存放物件並進行刪除

更新(Update)資料
n-7-10-updating-documents/playground/mongodb-update.js

//findOneAndUpdate(filter,update,options,callback)

//test 1.
db.collection('Todos').findOneAndUpdate({
    _id: new ObjectID('59ec320b05a4b74e3bdff8c1')
}, {
    $set: {
      age : 1
    }
}, {
    returnOriginal: false
}).then((result) => {
    console.log(result);
});

filter為篩選條件，像剛剛的find()方法一樣，裡頭篩物件
update放要更新的物件，如果搜尋不到則會增加新物件
options有多個參數方式，大部分使用returnOriginal:false，回傳更新後的物件
callback能夠丟入then(res)進一步呈現
